Install be-BOP in minutes

Run a single command on your server and the installer will do the rest. When it finishes, be-BOP will be ready for initial configuration.

Requirements

  • A Linux machine running Debian or Ubuntu LTS. Minimum: 2 GB RAM, 40 GB disk, internet access.
  • A domain name you control. Example: myapp.example.com
  • An email address used only for Let's Encrypt TLS certificates. We don't store or receive it.

How do I get this?

Generate your install command

Enter your details (for your own reference). The wizard will ask for them during setup.

No domain yet? You can register one with any registrar. After purchase, create an A record pointing to your server's IPv4 address. If your machine has an IPv6 address, also create an AAAA record. Learn how.

Run this on your server:

bash <(curl -fsSL https://be-bop.io/wizard/install.sh)

Prefer to review first? Open https://be-bop.io/wizard/install.sh in a browser to read the script before running it. The script will also display all planned tasks and ask for confirmation before making any changes to your system.

Installation typically completes in 3–10 minutes, depending on network speed and package updates.

After installation

Visit your domain in a browser. be-BOP will be ready for initial configuration via the web interface.

How to get what you need (domains, servers, DNS)

1) Get access to a server

You'll need access to a server where you can run commands. This can be a virtual private server (VPS) from any hosting provider, a dedicated machine, or even a spare machine at home running Linux, as long as it meets the requirements and has a public IP address reachable from the internet.

Once your server is ready, make sure you can connect to it:

  • Via SSH (using your terminal or an SSH client), or
  • Through your provider's web console, if they offer one.

To confirm that you have shell access, try running a simple command like: 

date

If it prints the current date and time, you’re in the right place.

You don’t need to choose a specific provider — any that give you control over a Debian or Ubuntu LTS environment with a minimum of 2 GB RAM, 40 GB disk, and internet access will work fine.

2) Get a domain name

  • Register a domain with any registrar you prefer. Popular choices work fine; be-BOP is provider-agnostic.
  • Decide on a hostname (e.g. yourdomain.tld or shop.yourdomain.tld).

3) Point DNS to your server

  • Create an A record for your hostname pointing to your server's IPv4 address. You can usually find this setting in your domain registrar or DNS provider's dashboard, often under sections called “DNS Management” or “Zone Editor.” The IPv4 address is a series of four numbers separated by dots, like 203.0.113.42.
  • If your server also has IPv6 (not all providers offer it), add an AAAA record. An IPv6 address looks different — for example, 2001:db8::1234. If your provider doesn’t list an IPv6 address for your server, you can safely skip this step.
  • TTL can be left at the default (e.g. 300–3600 seconds).
  • Wait for DNS to propagate (usually instantaneous for new domains; otherwise may take a few minutes, per the existing TTL).

4) Ready for takeoff

  • Once you have access to your server (see the “Get access to a server” section above), copy the installation command from this page, paste it into the terminal, and press Enter (or Return) to start the installation process.
What the installer does
  1. Environment checks — verifies OS, CPU/RAM/disk, network reachability, and required ports.
  2. TLS certificates — obtains and installs HTTPS certificates from Let's Encrypt automatically.
  3. Deployment — fetches be-BOP components, sets up services, configures networking and persistence.
  4. Bring-up — starts the web interface. At this point, be-BOP is ready for initial configuration.
Security & transparency notes
  • You can always review the installer at https://be-bop.io/wizard/install.sh before running it.
  • The domain and email you provide are used locally by the wizard for certificate provisioning and are not sent to be-BOP.
  • Run the command only on a machine you control and trust.
