Commencer

Installez be-BOP en quelques minutes

Exécutez une seule commande sur votre serveur et l'installateur fera le reste. Une fois terminé, be-BOP sera prêt pour la configuration initiale.

Prérequis

  • Une machine Linux sous Debian ou Ubuntu LTS. Minimum : 2 Go RAM, 40 Go disque, accès internet.
  • Un nom de domaine que vous contrôlez. Exemple : example.com
    Vous aurez besoin de deux enregistrements DNS pointant vers votre serveur : 
    example.com      → A / AAAA → IP de votre serveur
s3.example.com   → A / AAAA → IP de votre serveur
  • Une adresse email utilisée uniquement pour les certificats TLS Let's Encrypt. Nous ne la stockons ni ne la recevons.

Comment obtenir tout cela ?

Générez votre commande d'installation

Entrez vos informations (pour votre propre référence). L'assistant vous les demandera pendant la configuration.

be-BOP utilise votre domaine principal et un sous-domaine s3.<domaine>. Consultez la section Prérequis ci-dessus pour savoir quels enregistrements DNS créer.
Utilisé uniquement pour l'émission de certificats et les rappels de renouvellement de Let's Encrypt. Il n'est pas envoyé à be-BOP.

Exécutez ceci sur votre serveur :

bash <(curl -fsSL https://be-bop.io/wizard/install.sh)

Vous préférez examiner d'abord ? Lisez le script ici avant de l'exécuter. Le script affichera également toutes les tâches prévues et demandera confirmation avant d'apporter des modifications à votre système.

L'installation se termine généralement en 3 à 10 minutes, selon la vitesse du réseau et les mises à jour des paquets.

Après l'installation

Visitez votre domaine dans un navigateur. be-BOP sera prêt pour la configuration initiale via l'interface web.

Comment obtenir ce dont vous avez besoin (domaines, serveurs, DNS)

1) Obtenir l'accès à un serveur

Vous aurez besoin d'accéder à un serveur où vous pouvez exécuter des commandes. Cela peut être un serveur privé virtuel (VPS) de n'importe quel hébergeur, une machine dédiée, ou même une machine de rechange chez vous sous Linux, tant qu'elle répond aux exigences et dispose d'une adresse IP publique accessible depuis internet.

Une fois votre serveur prêt, assurez-vous de pouvoir vous y connecter :

  • Via SSH (en utilisant votre terminal ou un client SSH), ou
  • Via la console web de votre fournisseur, s'il en propose une.

Pour confirmer que vous avez accès au shell, essayez d'exécuter une commande simple comme : 

date

Si elle affiche la date et l'heure actuelles, vous êtes au bon endroit.

Vous n'avez pas besoin de choisir un fournisseur spécifique — tout fournisseur qui vous donne le contrôle sur un environnement Debian ou Ubuntu LTS avec un minimum de 2 Go de RAM, 40 Go de disque et un accès internet fonctionnera parfaitement.

2) Obtenir un nom de domaine

Enregistrez un domaine chez le fournisseur de votre choix. Vous pouvez utiliser votre domaine racine (ex. example.com) ou un sous-domaine (ex. boutique.example.com).

Vous configurerez le DNS à l'étape suivante.

3) Pointer le DNS vers votre serveur

Configurez le DNS pour que votre nom d'hôte choisi et son sous-domaine compagnon s3. pointent tous deux vers votre serveur :

  • Créer des enregistrements A (IPv4) 
    nom-hôte-choisi       → A → IPv4 de votre serveur
s3.nom-hôte-choisi    → A → IPv4 de votre serveur
    Vous pouvez généralement trouver ce paramètre dans le tableau de bord de votre registrar ou fournisseur DNS, souvent dans des sections appelées "Gestion DNS" ou "Éditeur de zone". L'adresse IPv4 est une série de quatre nombres séparés par des points, comme 203.0.113.42.
  • Si votre serveur a l'IPv6, créez des enregistrements AAAA 
    nom-hôte-choisi       → AAAA → IPv6 de votre serveur
s3.nom-hôte-choisi    → AAAA → IPv6 de votre serveur
    Une adresse IPv6 est différente — par exemple, 2001:db8::1234. Si votre fournisseur ne liste pas d'adresse IPv6 pour votre serveur, vous pouvez ignorer cette étape en toute sécurité.
  • Le TTL peut être laissé par défaut (ex. 300-3600 secondes).
  • Attendez la propagation DNS (généralement instantanée pour les nouveaux domaines ; sinon peut prendre quelques minutes, selon le TTL existant).

4) Prêt pour le décollage

  • Une fois que vous avez accès à votre serveur (voir la section "Obtenir l'accès à un serveur" ci-dessus), copiez la commande d'installation de cette page, collez-la dans le terminal, et appuyez sur Entrée pour démarrer le processus d'installation.

5) Où et comment obtenir de l'aide

Si quelque chose ne fonctionne pas comme prévu, ne vous inquiétez pas — nous pouvons vous aider à résoudre le problème. Lorsque vous nous contactez, veuillez inclure quelques détails pour que nous puissions comprendre votre configuration rapidement :

  • Le fournisseur d'hébergement où votre serveur fonctionne.
  • Le registrar de domaine que vous avez utilisé pour vos paramètres DNS.
  • Votre nom de domaine et les spécifications de base du serveur (une capture d'écran de votre page VPS montre généralement cela).
  • Une capture d'écran du message d'erreur et toute sortie visible, ou notez si rien n'est apparu.

Vous pouvez également inclure la sortie complète de la commande si vous avez déjà exécuté l'installation.

Ce que fait l'installateur
  1. Vérifications d'environnement — vérifie l'OS, CPU/RAM/disque, l'accessibilité réseau et les ports requis.
  2. Certificats TLS — obtient et installe automatiquement les certificats HTTPS de Let's Encrypt.
  3. Déploiement — récupère les composants be-BOP, configure les services, le réseau et la persistance.
  4. Démarrage — lance l'interface web. À ce stade, be-BOP est prêt pour la configuration initiale.
Notes de sécurité et transparence
  • Vous pouvez toujours examiner le script d'installation avant de l'exécuter.
  • Le domaine et l'email que vous fournissez sont utilisés localement par l'assistant pour l'approvisionnement des certificats et ne sont pas envoyés à be-BOP.
  • Exécutez la commande uniquement sur une machine que vous contrôlez et en laquelle vous avez confiance.
